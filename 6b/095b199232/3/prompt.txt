# Fix: ExceptionBadge SSE Connection Errors

## Context

The browser console on production (`https://secman.covestro.net`) shows 6 JavaScript errors from the ExceptionBadge SSE connection in the Sidebar. The pattern repeats: connect → receive initial count (0) → immediate connection error → exponential backoff retry (1s, 2s, 4s, 8s). After 5 failures, the badge gives up and stays at 0.

**Root cause**: After the backend sends the initial count event, the `sink.asFlux()` emits nothing (no exception count changes happening), so the connection goes idle. The corporate reverse proxy (nginx/F5/load balancer) detects the idle SSE stream and terminates it. The backend has **no heartbeat/keep-alive** mechanism to prevent this.

**Secondary issues**:
- No `X-Accel-Buffering: no` header (nginx buffers SSE responses by default)
- Frontend logs every retry as `console.error` (will fail the new Playwright E2E test from feature 081)
- Frontend gives up entirely after 5 errors instead of falling back to HTTP polling

## Changes

### 1. Backend: Add SSE heartbeat (primary fix)

**File**: `src/backendng/src/main/kotlin/com/secman/controller/ExceptionBadgeUpdateHandler.kt`

Add `import java.time.Duration` and modify `streamBadgeUpdates()`:
- Add `Flux.interval(Duration.ofSeconds(15))` emitting heartbeat events with name `"heartbeat"`
- Use `Flux.merge(heartbeat, countUpdates)` inside the existing `Flux.concat` so heartbeats interleave with real updates
- The frontend only listens for `count-update` events, so `heartbeat` events are silently ignored by EventSource
- Heartbeat uses `CountUpdateData(-1)` to satisfy the generic type constraint

```kotlin
val heartbeat = Flux.interval(Duration.ofSeconds(15))
    .map { Event.of(CountUpdateData(-1)).name("heartbeat") }

val countUpdates = sink.asFlux()
    .map { data -> Event.of(data).name("count-update") }

return Flux.concat(
    Flux.just(initialEvent),
    Flux.merge(heartbeat, countUpdates)
        .doOnCancel { ... }
        .doOnError { ... }
)
```

### 2. Backend: Add SSE proxy headers

**File**: `src/backendng/src/main/kotlin/com/secman/filter/SecurityHeadersFilter.kt`

After the `/api/` cache-control block (line 122), add SSE-specific headers for the two SSE endpoints:

```kotlin
if (request.uri.path == "/api/exception-badge-updates" ||
    request.uri.path == "/api/materialized-view-refresh/progress") {
    response.header("X-Accel-Buffering", "no")
    response.header("Connection", "keep-alive")

```
