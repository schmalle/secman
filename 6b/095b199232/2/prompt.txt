# Plan: Replace Domain Text Input with Select Dropdown in User Mapping Form

## Context

When editing a user's access mappings in the admin user management page, the Domain field is currently a free-text input. This forces admins to remember and manually type domain names. The goal is to replace it with a `<select>` dropdown populated with all existing domains from the system, so admins can pick from known domains.

## Approach

Create a new backend endpoint that returns all distinct domains (merged from both `user_mapping` and `asset` tables), then use it to populate a Bootstrap `<select>` in the frontend form.

## Changes

### 1. Backend: Add `findDistinctDomains()` to UserMappingRepository
**File:** `src/backendng/src/main/kotlin/com/secman/repository/UserMappingRepository.kt`
- Add JPQL query: `SELECT DISTINCT m.domain FROM UserMapping m WHERE m.domain IS NOT NULL AND m.domain != ''`
- Follows existing pattern of `findDistinctDomainByEmail` (line 124)

### 2. Backend: Add `getDistinctDomains()` to UserMappingService
**File:** `src/backendng/src/main/kotlin/com/secman/service/UserMappingService.kt`
- Merge domains from `userMappingRepository.findDistinctDomains()` + `assetRepository.findDistinctAdDomains()`
- Lowercase, deduplicate, sort
- Inject `AssetRepository` if not already present

### 3. Backend: Add `GET /api/user-mappings/domains` endpoint
**File:** `src/backendng/src/main/kotlin/com/secman/controller/UserMappingController.kt`
- `@Get("/domains") @Secured("ADMIN")` returning `List<String>`
- No conflict with existing routes (`/current`, `/applied-history`, `/{id}`)

### 4. Frontend: Add `getDomains()` API function
**File:** `src/frontend/src/services/userMappingService.ts`
- Call `authenticatedGet('/api/user-mappings/domains')` and return `string[]`

### 5. Frontend: Replace text input with `<select>` in IpMappingForm
**File:** `src/frontend/src/components/IpMappingForm.tsx`
- Add state: `domains: string[]`
- Fetch domains on mount via `useEffect`
- Replace `<input type="text">` with `<select className="form-select">` with `-- No domain --` default option
- Handle edge case: if editing and current domain not in list, include it

## Verification
1. `./gradlew build` — backend compiles with no errors
2. Run the app, navigate to Admin > User Management, edit a user
3. In the Access Mappings section, click Add New Mapping or edit existing
4. Domain field should show a dropdown with existing domains
5. Selecting a domain and saving should work as before
